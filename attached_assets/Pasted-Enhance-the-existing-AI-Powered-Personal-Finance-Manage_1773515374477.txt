Enhance the existing AI Powered Personal Finance Manager application by implementing a complete User Profile & Account Settings Module.

This feature must integrate cleanly with the current architecture:

Backend:
Node.js + Express + JWT Auth + PostgreSQL (Drizzle ORM) + Redis

Frontend:
React + Tailwind + Recharts

AI:
OpenAI based financial advisor and categorization services

Do NOT break existing APIs or UI flows.

------------------------------------------------
DATABASE CHANGES
------------------------------------------------

Update users table:

Add columns:

avatarUrl TEXT NULL
lastLoginAt TIMESTAMP NULL
savingsTargetPercent INT DEFAULT 20
riskAppetite VARCHAR(20) DEFAULT 'MEDIUM'
defaultCurrency VARCHAR(10) DEFAULT 'INR'
notificationBudget BOOLEAN DEFAULT true
notificationSubscription BOOLEAN DEFAULT true
notificationAI BOOLEAN DEFAULT true
isDeleted BOOLEAN DEFAULT false

------------------------------------------------
BACKEND IMPLEMENTATION
------------------------------------------------

Create new controller: userController
Create service: userService

1. Get User Profile

GET /user/profile

Return:

{
 id,
 name,
 email,
 role,
 avatarUrl,
 createdAt,
 lastLoginAt,
 savingsTargetPercent,
 riskAppetite,
 defaultCurrency,
 notificationSettings
}

2. Update Profile

PUT /user/profile

Allow update:

- name
- avatarUrl
- defaultCurrency

Validate inputs.

3. Change Password

PUT /user/change-password

Body:

{
 currentPassword,
 newPassword
}

Verify BCrypt match.

4. Update Financial Preferences

PUT /user/preferences

Fields:

- savingsTargetPercent (0-100)
- riskAppetite (LOW / MEDIUM / HIGH)

5. Notification Preferences

PUT /user/notifications

Toggle:

- budget alerts
- subscription alerts
- AI advice alerts

6. Financial Snapshot API

GET /user/financial-summary

Return:

{
 currentBalance,
 totalMonthlySpending,
 activeSubscriptions,
 financialHealthScore
}

Reuse existing services (transactions / subscriptions / AI health score).

Cache response in Redis:

financialSummary:userId
TTL = 5 minutes

Invalidate cache on:

- new transaction
- subscription change
- health score recompute

7. Activity Log (Optional but recommended)

Create table:

user_activity

Fields:

id
userId
action
device
ipAddress
createdAt

Log:

- login
- password change
- profile update

API:

GET /user/activity
