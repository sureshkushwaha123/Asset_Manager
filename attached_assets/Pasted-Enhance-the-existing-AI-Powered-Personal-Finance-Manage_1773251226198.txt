RECURRING DETECTION LOGIC
--------------------------------------------

When a new transaction is created:

Run a background detection service.

Steps:

1. Find transactions of same user where merchantName similarity > 80%.
2. Sort by date.
3. Check if at least 3 transactions exist with:

   - interval between 25–35 days (monthly cycle tolerance)
   - amount variation < 15%
   - same debit type

4. If conditions match:

   Mark latest transaction:

   isRecurring = true
   recurringCycle = "monthly"

5. Calculate:

confidenceScore based on:

- interval consistency
- amount consistency
- number of repetitions

6. Estimate nextExpectedDate:

lastTransactionDate + detectedCycleInterval

7. Insert or update record in subscriptions table.
