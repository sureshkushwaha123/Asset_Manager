1. Authentication

Endpoints:

POST /auth/register
POST /auth/login

Password must be encrypted using BCrypt.

Include role based authorization.

--------------------------------------------------

2. Transaction Management

Create REST APIs:

POST /transactions
GET /transactions
GET /transactions/filter?type=DEBIT
GET /transactions/filter?dateRange=
GET /transactions/summary

Features:
- add transaction
- filter by date
- filter by debit/credit
- pagination

--------------------------------------------------

3. AI Powered Categorization

When a transaction is created:

Example input
description = "Netflix"

AI should categorize it automatically.

Example response
category = "Entertainment"

Use Spring AI with OpenAI OR Ollama.

Prompt example:
"Classify this transaction into a financial category: Netflix"

Possible categories:
Food
Travel
Bills
Entertainment
Subscription
Shopping
Other

--------------------------------------------------

4. AI Financial Advisor Chatbot

Create endpoint

POST /ai/ask

Example prompt from user:

"How much did I spend on coffee this month and can I afford a new phone?"

Steps:

1 Query user's transactions from PostgreSQL
2 Format the data
3 Send to LLM using prompt engineering
4 Return financial advice

Ensure the AI only sees that user's data.

--------------------------------------------------

5. Budget System

Users can define monthly budgets per category.

Example

Food = ₹5000

When spending reaches:

80% → send warning
100% → send alert

Store alerts in database.

Use Redis to cache:
